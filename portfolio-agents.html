Agent Architecture

Seven agents, one cohesive picture

Each agent has a dedicated role, data sources, and deterministic skill tools that guarantee accurate computation. The LLM reasons about what the numbers mean — the tools handle the math. CrewAI orchestrates execution and passes context so agents can cross-reference everything into actionable intelligence.

📈
Portfolio Analyst
Agent 1 · Performance, Risk & Correlation

Evaluates all holdings with five deterministic skill tools: Sharpe ratio and volatility calculations, concentration risk detection (HHI, sector limits), allocation drift vs. targets, performer ranking with percentiles, and pairwise correlation analysis to uncover hidden concentration across holdings that appear diversified.

Agent 2 · Congressional Trading Signals

Under The Hood

How agents think

Every agent follows a "tools compute, LLM reasons" architecture. Deterministic Python tools read full article text, score severity, map portfolio impact, and pack a compact briefing — so the language model only receives structured results it can reason about, not raw data it has to parse.

📡
Step 1
Fetch from multiple sources

Each tool pulls data from every relevant source — NewsAPI for broad coverage, Finnhub for market-specific feeds, Yahoo Finance for earnings calendars. Articles are fetched at full length, then deduplicated by headline before any analysis begins.

⚖️
Step 2
Multi-signal sentiment scoring

Each article is scored by three independent systems: custom keyword rules for speed (54 financial keywords, severity 1–5), NLTK's VADER model for NLP nuance (7,500-word lexicon, understands negation and intensifiers), and Alpha Vantage's AI model for financial-domain depth (per-ticker relevance scores). The CNN Fear & Greed Index adds a market-wide sentiment reading. When signals disagree — alarming headline, reassuring substance — that's flagged as the most interesting insight.

🏷️
Step 3
Categorize & detect events

Articles are classified into macro themes (Fed policy, inflation, trade, geopolitical) and material event types (earnings, analyst actions, insider trades, regulatory, corporate). Each article can trigger multiple categories — a Fed rate decision article might tag both fed_policy and inflation.

🎯
Step 4
Map portfolio impact

A static impact map connects macro categories to specific portfolio holdings. A trade policy headline automatically links to emerging market and tech supply chain tickers. The tool calculates total portfolio exposure as a percentage — so "trade war" isn't abstract, it's "38% of your holdings."

📦
Step 5
Rank, pack & trim

Articles are ranked by severity × portfolio relevance, then only the top stories make the cut. Raw text is discarded — what survives is a compact JSON briefing with scores, tickers, and rationales. Hundreds of articles become ~2K tokens of structured signal.

🧠
Step 6
Claude reasons across all signals

The language model receives the pre-scored briefing packet with every signal layered in: keyword severity, VADER NLP, Alpha Vantage AI scores, Fear & Greed context, and signal disagreement flags. Its job is synthesis: what does this pattern of signals mean for timing, risk, and action? Four independent reads on each article, one reasoning step to connect them.

💡
Why this architecture matters

Traditional agent designs send raw data to the LLM and let it figure things out — expensive, slow, and token-hungry. This system layers four independent sentiment signals (keyword rules, VADER NLP, Alpha Vantage AI, Fear & Greed Index) before the LLM ever sees the data. The Python tools read thousands of words and produce structured, multi-signal scores for free; Claude reads a compact briefing and reasons across all signals — like a senior analyst receiving a research brief from four different desks.

Global account switcher scopes all pages to a specific brokerage account. React Context architecture means every dashboard view becomes account-aware without per-page refactoring.

🧪
Multi-Signal Sentiment

Four independent sentiment signals — keyword rules, VADER NLP, Alpha Vantage AI, CNN Fear & Greed — scored before the LLM sees any data. Signal disagreements are flagged as the most interesting insights.

Tech Stack

Built with Python + React

🤖
CrewAI
Agent Orchestration

Role-based multi-agent coordination with deterministic skill tools. Agents reason with Claude; tools handle the math.

🧠
Claude (Anthropic)
LLM Reasoning Engine

Claude Sonnet powers each agent's analysis, classification, and narrative generation via the Anthropic Python SDK.

⚡
FastAPI + Uvicorn
Backend API

Async REST API with JWT authentication, rate limiting, security headers, and CORS. Deployed on Railway.

⚛️
React 18 + Vite
Frontend

Single-page app with React Router, Tailwind CSS, Recharts for portfolio visualizations, and Lightweight Charts for candlesticks. Deployed on Vercel.

📈
Finnhub + SnapTrade
Market Data + Brokerage

Finnhub for live quotes, fundamentals, news, and congressional trades. SnapTrade for full brokerage access — holdings, balances, transactions, orders, performance, and multi-account support. yfinance as fallback.

⚖️
Multi-Signal Sentiment
NLP + AI Scoring

VADER NLP (7,500-word lexicon), Alpha Vantage AI sentiment (per-ticker relevance), CNN Fear & Greed Index (market-wide context), and custom keyword rules. Four reads per article before the LLM reasons.

🗄️
SQLite + SQLAlchemy
Encrypted Storage

SQLAlchemy ORM with WAL mode, parameterized queries, and optional Fernet encryption at rest for financial data.

🔐
Security Stack
Auth, Encryption & Audit

JWT auth (HS256), macOS Keychain for secrets, Fernet encryption at rest, structured audit logging, rate limiting, and security headers.

🎙️
ElevenLabs
Podcast Audio (Phase 6)

Two-voice text-to-speech API that will convert analysis scripts into podcast-style audio briefings.
